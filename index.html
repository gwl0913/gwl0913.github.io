发表于 2020-07-24 更新于 2020-07-29

从”三分之一”说起

分数1/3转换为小数,其表达形式为0.333...
其中的...表示后面跟着无穷多个3. 
    这意味着计算会一直进行.不会有有精确结果.
前面的转换是十进制下进行的. 如果采用三进制,则转换结果就是0.1
    这意味着在三进制下,转换结果是精确的.
下面的表格对比更清楚的说明这些情况

表达有差异 表达,性质有差异 表达有差异 表达,性质有差异
小数位无穷多 小数位只有1位
无精确值 有精确值

关于无穷

十进制下的0.333..., 三进制下的0.1两者是同一个数,但前者具有无穷多的小数位,后者却只有1位小数位.
这表明:
    1. 同一事物是否无穷性,取决于考察该事物时所采用的规则.
    2. 无穷是特定规则下的产物,其本身不具有公理性.

“1”的特殊意义

十进制下的0.333...和三进制下的0.1,尽管表达形式和性质都有非常大的不同,但却是同一个数.
其中的原因是:不同进制下的数可以相互转换.能够进行这种相互转换的基础是: 在不同进制间有一个相同的标准.
    这个标准就是"1". 不论在何种进制下,"1"的意义都是一样的.
    就是说"1"有进制不变性.

“1”与光速的相似性

与1的这种不变性相似,物理学中的光速也有不变性.
光速和参考系无关,不论在哪个参考系,光速总是30万公里每秒.
    1的不变性和光速的不变性之间是否存在必然关系呢?

用好promise的关键

  1. 把握好 .then() 和 .catch().

    1. .then() 和 .catch()都接收一个参数, 这个参数是 一个单参数的回调函数.
    2. 第一个返回promise呼叫的是起始,后面可以跟随任意多的.then().
    3. 之后的.then()都会依次执行
    4. 能够执行到最后一个.then() 表示一切顺利
    5. .catch()负责处理夭折通知. 注意 夭折可以发生在任何阶段.

  2. 每一个.then()中的回调函数,都需要返回promise.

  3. promise的本质是把多个异步处理串联起来, 形成一个大的异步处理

封装路由功能

路由功能是处理AJAX请求, 动态网站处理的基础， 所以现在的web服务框架路由功能是标配. 下面我们为myExpress包封装路由功能. 并让其路由功能和express框架的使用方式一致.

思路和开发过程规划

  1. 添加路由配置保存场所.
  2. 添加路由配置入口
  3. 添加路由处理
    1. 调整服务入口,优先处理路由,后处理静态资源请求.
    2. 扩展 nodejs 原生的 request 和 response. 添加express框架具备的功能

开发过程

添加配置项 添加路由配置保存场所. 为get请求和post请求提供独立的保存场所, 预备应对相同的请求路径, 不同的请求方式的情况

添加路由配置入口 应用程序通过这个入口,自己配置路由. 当然配置接口也分为 get 路由接口和 post 路由接口

调整服务入口 使得服务既可以处理静态资源请求.又可以处理动态的路由. 并优先处理动态路由.

封装路由处理逻辑 这段逻辑分为3段,

  1. 扩展response. 为其添加 send 功能,
  2. 扩展 request, 为其添加 query 属性.
  3. 呼叫用户指定的路由响应.

区别对待get,post get请求和post请求的传参模式不同, 他们的参数提取方式也不同. 而且post请求的参数提取过程比较繁琐.

处理post路由 在nodejs中post请求的参数是异步的流形式传输. 通过监控, data,end 事件,可以获取请求参数

调整处理静态资源 主要是把默认主页的处理纳入静态资源处理过程.
