添加静态网站功能

​ 现在 myExpress 只能回应 “我们的web服务框架(包)正在开发中“.下面 添加静态网站挂载功能. 添加了静态网站挂载功能后, 可以根据浏览的地址来回应不同的的网页. 这些网页是一个目录结构下的各类静态网页资源. 我们的目标是: 只要把html, css,js ,img 等资源文件放置在特定的目录下就可以,通过浏览器远程浏览这些网页资源. 不论资源的类型是多么独特, 不论这些资源的目录结构有多复杂, 都可以浏览访问.

思路

  1. 提取请求的文件扩展名
  2. 根据扩展名构造回应头， 关键是头中的mime类型
  3. 根据请求路径打开静态资源文件，把文件内容作为回应的主体

封装路由功能

路由功能是处理AJAX请求, 动态网站处理的基础， 所以现在的web服务框架路由功能是标配. 下面我们为myExpress包封装路由功能. 并让其路由功能和express框架的使用方式一致.

思路和开发过程规划

  1. 添加路由配置保存场所.
  2. 添加路由配置入口
  3. 添加路由处理
    1. 调整服务入口,优先处理路由,后处理静态资源请求.
    2. 扩展 nodejs 原生的 request 和 response. 添加express框架具备的功能

开发过程

添加配置项 添加路由配置保存场所. 为get请求和post请求提供独立的保存场所, 预备应对相同的请求路径, 不同的请求方式的情况

添加路由配置入口 应用程序通过这个入口,自己配置路由. 当然配置接口也分为 get 路由接口和 post 路由接口

调整服务入口 使得服务既可以处理静态资源请求.又可以处理动态的路由. 并优先处理动态路由.

封装路由处理逻辑 这段逻辑分为3段,

  1. 扩展response. 为其添加 send 功能,
  2. 扩展 request, 为其添加 query 属性.
  3. 呼叫用户指定的路由响应.

区别对待get,post get请求和post请求的传参模式不同, 他们的参数提取方式也不同. 而且post请求的参数提取过程比较繁琐.

处理post路由 在nodejs中post请求的参数是异步的流形式传输. 通过监控, data,end 事件,可以获取请求参数

调整处理静态资源 主要是把默认主页的处理纳入静态资源处理过程.

使用vscode的正确姿势

搭建 vscode 和virtualBox 配合的集成开发环境
搭建完成后, 使用远程机器和使用本机一样方便
核心是 vscode 的 Remote Development 插件的安装
通过这个插件 vscode 管理虚拟机文件和管理本机文件没有什么差别

安装基础工具

需要在本机安装 4 个基础工具. Xshell vscode git virtualBox
这 4 个工具都可以通过 360软件管家 方便的安装

Xshell 远程登陆工具. 用来连接虚拟机.

Xshell 是一个强大的安全终端模拟软件，它支持 SSH1, SSH2, 以及 Microsoft Windows 平台的 TELNET 协议。Xshell 通过互联网到远程主机的安全连接以及它创新性的设计和特色帮助用户在复杂的网络环境中享受他们的工作。

vscode 代码编写 工具 添加插件后功能媲美IDE

Visual Studio Code（VS Code）近年来获得了爆炸式增长，成为广大开发者工具库中的必备神器。它作为一个开源项目，也吸引了无数第三方开发者和终端用户，成为顶尖开源项目之一。它在功能上做到了够用，体验上做到了好用，更在拥有海量插件的情况下做到了简洁流畅，实属难能可贵。

git 版本控制工具

Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

virtualBox 虚拟机软件

VirtualBox（vbox虚拟机）是德国一家软件公司InnoTek 所开发的虚拟系统软件。虚拟机软件virtualbox不仅具有丰富的特色，而且virtualbox虚拟机的性能也很优异。

原型虚拟机安装篇

装好virtualBox后系统会多出一块虚拟网卡. 这个网卡是用来和虚拟机通讯的. 这个网卡默认的地址是 192.168.56.1
以后创建的虚拟机就可以在 192.168.56.xxx 这个网段来构建一个局域网.

virtualBox 配置

​ 管理=>全局设定=>默认虚拟电脑位置 配置一个不怕空间浪费的地方 ( 其实也用不了多少 )
